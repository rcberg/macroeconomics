# The Neoclassical Model of Location, Albouy and Stuart (2020)

The Neoclassical Model of Location, as set up by Albouy and Stuart ([doi](https://doi.org/10.1111/iere.12419), [working paper](https://bryan-stuart.com/files/UrbanPopulation_Feb2019.pdf)), is a general equilibrium model with 2 output sectors-- one traded good and one home (housing) good-- as well as 3 factors of production-- land, labor, and capital. The full nonlinear model is shown in Supp. Appendix B and C. Because Julia has very powerful libraries for solving all kinds of systems of equations, I decided to sharpen my skills and write a function that solves the model for the 15 endogeneous variables, subject to 15 equilibrium conditions.

-    [The Execution script](https://github.com/rcberg/macroeconomics/blob/main/scripts/neoclassical-location-model/symbolic_function_city_simulator_hessian_exe.jl) combines the two helper function scripts (symbolic_function_city_func_compiler.jl and symbolic_function_city_builder.jl) to solves the economic model they produce. It comes pre-loaded with a flow that matches the Albouy and Stuart (2020) large city/small city nonlinear model. The helper functions are separated so that anyone can tweak the cost and expenditure functions as desired, without disturbing the other. This project benefited from [Claude (Opus 4.6)](https://claude.ai/) to help bridge Symbolics.jl and JuMP.jl libraries.
-    [The Compile script](https://github.com/rcberg/macroeconomics/blob/main/scripts/neoclassical-location-model/symbolic_function_city_simulator_func_compiler_hessians.jl) containing the helper function that uses Symbolics.jl to take user-defined cost and expenditure functions and calculate their gradient vectors, Jacobian matrices, and Hessian matrices. It compiles them into useable Julia functions to be used by the optimizer, with help from the "builder"" function. **This is the script to modify if you want to supply alternative cost/expenditure functions.**
-    [A Builder script](https://github.com/rcberg/macroeconomics/blob/main/scripts/neoclassical-location-model/symbolic_function_city_simulator_builder_hessians.jl) containing the helper function which builds the JuMP model of the nonlinear system. It takes the compiled cost and expenditure functions and sets up the nonlinear system from Appendix B and C from the paper, which gets fed to the Ipopt optimizer to solve. The optimization is more efficient because the exact Hessian matrix is provided to Ipopt rather than relying on an approximation, so the optimizer converges faster and more accurately. Following the 'MPEC' method, the equilibrium conditions are expressed as constraints. The model is calibrated by minimizing a loss function in the squared deviation of the CES parameters from their Table 1 targets, subject to the equilibrium constraints. The important large city values can then be supplied for the small city, whose population and local prices are endogeneously determined.

-    [This](https://github.com/rcberg/macroeconomics/blob/main/scripts/neoclassical-location-model/large_city_small_city_simulator_functions.jl) is the legacy script which the above workflow is based off of. It technically ends up 0.0008021599516072619\% closer to the Table 1 values, because the legacy version of Ipopt it uses efficiently calculates exact first and second derivatives, but it takes 69% more time per run (0.535s vs 0.316s) and about 17% more iterations (142 vs 121).

- An example of using this model for some comparative statics, showing the differences between tax regimes, can be found [here.](https://github.com/rcberg/macroeconomics/blob/main/scripts/neoclassical-location-model/large_city_small_city_tax_comparisons.jl) The workhorse script for this part is a version of the above script which has been modified to accept user-defined parameters; find that [here.](https://github.com/rcberg/macroeconomics/blob/main/scripts/neoclassical-location-model/large_city_small_city_simulator_taxsim.jl)
